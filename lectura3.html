Lineamientos de desarrollo de proyectos web

Las aplicaciones deben realizarse atendiendo el marco que aplique a cada desarrollo dentro de lo que establece el estándar de seguridad de Aplicaciones Web denominado OWASP, mediante el cual se publica y mantiene actualizado tanto una guía para el desarrollo de aplicaciones seguras, como un listado de los diez riesgos de seguridad más importantes sobre los cuales recomienda a las organizaciones orientar la prevención y protección en el código que contienen sus aplicaciones.

1. Prefijos y Sufijos:


Es obligatorio utilizarlos para denominar cada tipo de elemento identificado como un artefacto de desarrollo, su función es identificar diferentes tipos de objetos, datos, entradas o salidas. Se puede aplicar para:

• Directorios y estructura de archivos.
• Archivos
• Clases, Interfaces, Servicios, Formularios, Apis, Enumeraciones, Etc.
• Proyectos, Soluciones, Espacios de Nombres.

2. Nomenclatura Base de Datos:


El nombre de la base de datos debe venir precedido del prefijo BDS (Base de datos) y seguido por el carácter ( _ ) underline. El nombre debe reflejar de manera precisa y exacta el contenido y su función. La nomenclatura utilizada es Pascal_Snake_Case:. No se deben utilizar espacios en el nombre de archivo.

3. Nomenclatura de Programación:


Se deben definir las variablesbr

• Definir funciones.
• Definir procedimientos.
• Definir clases.
• Definir objetos.

4. Accesibilidad y Usabilidad:


Siguiendo los Lineamientos y metodologías de Usabilidad para Gobierno en línea. Se debe garantizar el acceso universal a la Web, independientemente del tipo de hardware, software, infraestructura de red, idioma, cultura, localización geográfica y capacidades de los usuarios para lo cual se debe tener en cuenta las siguientes recomendaciones:

1. Dieseño centrado en el usuario.

2. Directrices de usabilidad web.
a. Objetivos del portal web.
b. Navegación global consistente.
c. url limpios.

3. Diseño de interfaz de usuario.
a. Ubicación del logotipo.
b. Interfaces en movimiento.
c. Contraste en brillo y color.
d. Información trasmitida a treves del color.
e. Justificación del texto.
f. Ancho del cuerpo de texto.
g. Desplazamiento horizontal.
h. Hojas de estilo para diferentes formatos.
i. Independencia de navegador.
j. Calidad del código: HTML Y CSS.

4. Diseño de interacción.
a. Campos obligatorios.
b. Asociación de etiquetas y campos.
c. Validación dinámica de datos.
d. Error de página no encontrada.
e. Ejemplos en los campos de formulario.
f. Páginas de confirmación.

5. Contenido.
a. Listas.
b. Escaneado de contenido.
c. Vínculos rotos.

5. Interoperabilidad:


El modelo de interoperabilidad de Colombia está diseñado para apoyar la implementación de la Estrategia de Gobierno en línea (GEL) que desarrolla el Ministerio TIC (Tecnologías de Información y las Comunicaciones), a partir de cuatro componentes: TIC para la gestión, TIC para servicios, TIC para el gobierno abierto y Seguridad y privacidad de la información.

6. Requisitos Técnicos:


Norma Técnica ISO/IEC 23270:2006 especifica la forma y establece la interpretación de los programas escritos en el lenguaje de programación C#.
Norma Técnica Colombiana NTC 5854, Accesibilidad a Páginas Web. C# Language Specification ECMA INTERNATIONAL Standard ECMA-334 4th edición – Junio 2006 ver: https://www.iso.org/obp/ui/es/#iso:std:iso-iec:23270:ed-2:v1:en
